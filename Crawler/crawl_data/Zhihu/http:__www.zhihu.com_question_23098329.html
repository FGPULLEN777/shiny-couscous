    Android 的系统字体 Roboto 设计如此。

    这个「问题」（虽然我更倾向于这是一种特性而非问题）很多无衬线字体都具有，如 Helvetica (Neue), Myriad, 等等。

    也有无衬线字体，如 Segoe UI 和 Verdana 等，为了字母的识别度，在大写 i 的上下两端加上两道衬线。但许多字体出于观感考虑，并不如此处理。

    实际上，大写 i 和小写 L 通常都会在一个 context 中出现，在阅读中混淆的机会比较少。因此，出于美观考虑，采用相同（或是相似，不少字体的这两个字符在高度等细节上还是有所区分的，如 Myriad 拼写 Illustration 一词时能看出头三个字母的区别）的单竖直线，亦是能够接受的。

    另外，也有无衬线字体对小写 L 进行处理，在尾部加上小弧线。如 Din, Trebuchet MS, Johnston 等等。这样的处理方法，既避免大写 i 加衬线导致的不美观，亦为字体增添一分特性，同时也继承手写的笔画特征。至于是否采取这种处理手法，只是字体设计师的个人选择。
