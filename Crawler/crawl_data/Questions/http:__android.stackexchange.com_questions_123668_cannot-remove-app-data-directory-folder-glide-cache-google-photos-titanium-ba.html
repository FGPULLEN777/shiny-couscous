Cannot remove app data directory folder glide_cache, Google Photos / Titanium Backup filesystem confusion

up vote 0 down vote favorite

I did a clean install of an updated KitKat 4.4.4 for my Atrix HD. I reinstalled all of my apps from Titanium Backup and everything else works fine. I did install an older version of gapps from before the Google Photos split from G+, and restored the newer Google Photos app from TiBu. When opening Google Photos it was unable to retrieve any auto backup settings. I uninstalled the app and tried installing from Google Playstore, but it had error "-24". I then looked at the /data/data directory as root to see what was going on:

I've since tried going into recovery and doing the same thing, as well as clearing cache/dalvik cache to no avail. If I try to install again from Titanium Backup, it hangs.
