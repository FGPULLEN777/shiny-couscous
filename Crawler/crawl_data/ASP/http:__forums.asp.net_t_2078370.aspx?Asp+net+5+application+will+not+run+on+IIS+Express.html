    hugovermaak

    I am just trying to debug, just hitting F5 on my keyboard. Do I honestly need to go through all that rig-marol just to debug?

    This is simply the change in the hosting platform (previously IIS was the defacto-standard) but now .NET 5 applications default to Kestrel, which will require you to install the HttpPlatformHandler found here.

    Once you have this installed locally, you should be able to go into your project.json file and explicitly set that you want to use it :

    "commands": {        "web": "Microsoft.AspNet.Hosting --server Microsoft.AspNet.Server.Kestrel",        "ef": "EntityFramework.Commands"},

    Then select the "web" option that appears within the DropDown menu in Visual Studio. This should launch a command line and will present you with the appropriate location where your application is being hosted (e.g. "localhost:5000")

    your web.config needs to be correct inside the wwwroot folder. Check this:

    <?xml version="1.0" encoding="utf-8"?><configuration>  <system.webServer>    <handlers>      <add name="httpPlatformHandler" path="*" verb="*" modules="httpPlatformHandler" resourceType="Unspecified" />    </handlers>    <httpPlatform processPath="%DNX_PATH%" arguments="%DNX_ARGS%" forwardWindowsAuthToken="false" startupTimeLimit="3600" />  </system.webServer></configuration>

    Also check that your running in rc1 and not an older version.

    Your application should have a logs directory with some additional information about issues with your application. Previously, you might have used something like the Event Viewer to check this, but with Kestrel, things are a bit "removed" from Windows and thus a log is usually your best bet.

    You can check the Event Viewer still to see if there might be an issue with IIS not having permission to write to the logs directory or something as well.
